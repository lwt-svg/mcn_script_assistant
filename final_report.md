## 1. 博主基础信息
- 主页链接：https://xhslink.com/m/1lYov1q8vUj

- 内容方向：精致一人食brunch｜氛围感酸奶碗教程｜居家简餐｜居家下午茶美食短视频

  - ③ 宅家下午茶｜简单又治愈的简餐合集

## 2. 人设、内容场景、表达方式、品牌匹配度
### 2.1 人设
松弛感居家美食爱好者，并非高强度自律减脂博主。主打慢悠悠在家制作简单精致的一人食，传递治愈的居家生活情绪，属于**真实自用分享型美食创作者**，无焦虑式身材说教。

### 2.2 内容场景
场景高度集中于居家厨房：周末宅家早餐、居家brunch、午后简易下午茶、5‑10分钟快手轻食简餐。几乎无外出探店内容，全部为居家操作台实拍。

### 2.3 表达方式
全程**无口播、无人声配音**，仅搭配舒缓轻音乐BGM叙事。
依靠**分段弹出的短字幕**传递文案信息，字幕克制简短，不做大段营销说教；
镜头以俯拍、食材特写、慢节奏轻微推拉运镜为主，依靠画面氛围感传递情绪，极大弱化广告痕迹。

### 2.4 品牌匹配度
1. **卖点匹配**：轻醒0蔗糖高蛋白酸奶主打低负担，适配早餐、下午茶、轻食加餐，完美契合博主制作酸奶碗、居家简餐的固定内容场景。
2. **调性匹配**：视频安静松弛、简约高级，与品牌拒绝极端减脂焦虑、健康轻生活的定位保持一致。
3. **植入可行性**：每期视频均会摆放多种食材进行摆盘拍摄，酸奶可以自然放置操作台、餐盘侧边出镜，依靠字幕软性输出卖点，实现无痕自然种草，规避硬广。
4. **人群匹配**：核心粉丝为24‑36岁追求精致健康饮食的都市女性，与品牌22‑35岁目标人群高度重合。

## 3. 内容完整拆解（结构｜钩子｜镜头｜表达范式）
> 单条视频标准时长：50‑60s
### 3.1 视频叙事结构
1. **开场钩子【0‑6s】**
镜头：俯拍干净操作台，平铺展示本次全部食材
字幕：周末宅家｜5分钟搞定治愈酸奶碗✨
> 钩子逻辑：营造松弛美好的居家场景，依靠氛围感抓住用户，不制造身材焦虑。

2. **食材展示 + 产品引入【6‑15s】**
镜头：依次给到食材特写，给到酸奶瓶身特写
字幕分段弹出：
> 最近很爱这款0蔗糖高蛋白酸奶
> 配料干净，很适合做减脂期加餐
> 作用：自然带出产品，轻量输出核心卖点，不强行推销。

3. **分步制作流程【15‑40s】**
镜头：手部操作特写，依次拍摄倒酸奶、铺水果、撒坚果，交替切换俯拍、45°侧拍
字幕仅客观描述操作动作：
> 铺一层醇厚酸奶
> 放上新鲜蓝莓
> 撒一把香脆坚果
> 仅记录制作过程，不加夸张营销话术。

4. **成品摆盘强化【40‑52s】**
镜头：45°斜拍完整酸奶碗成品特写，酸奶瓶与成品同框入镜
字幕补充产品价值：
> 每杯高蛋白≥8g
> 清爽不腻，解馋无负担

5. **结尾定格收尾【52‑60s】**
镜头：成品画面静态定格2‑3秒
字幕氛围感总结：简单治愈，好好享受一餐
> 结尾不引导下单，回归生活感受，降低观众广告抵触心理。

### 3.2 镜头范式
- 光线：柔和居家自然光，画面干净通透，低饱和治愈色调
- 机位：操作台俯拍、45°斜侧拍、手部操作特写、成品特写
- 运镜：极慢轻微推拉，无快速晃动，全程舒缓安静
- 音频：轻柔治愈纯音乐BGM，**全程禁止添加任何口播人声**

### 3.3 表达范式
1. 字幕使用短句，控制在2行以内，随镜头分段弹出，拒绝大段长文案
2. 产品卖点完全依靠字幕输出，画面只展示做饭流程，画面、字幕分工配合
3. 禁用“狂瘦”“掉秤神器”等夸大话术，仅客观描述口感、配料、食用场景
4. 叙事优先营造生活氛围，种草作为支线自然穿插于视频之中
5.  所有产品植入均为「场景化出现」，不做强行推荐，避免用户反感

## 4. 选择该博主作为Agent参考的理由
1. 整套视频范式高度标准化，镜头顺序、字幕位置、叙事流程固定，AI脚本智能体可以稳定复刻，不容易输出生硬带货脚本。
2. **无口播+字幕种草**形式差异化极强，区别于市面上大量口播减脂vlog，视频质感高级。
3. 场景与酸奶产品强绑定，智能体可批量生成早餐、下午茶、夏日加餐等不同主题短视频脚本，可复用性强。
4. 完全满足商单Brief约束：自然种草，拒绝硬广，不承诺减肥功效，输出脚本可直接交付达人进行真实拍摄。

### 3. 内容完整拆解（结构｜钩子｜镜头｜表达范式）
> 单条视频标准时长：35-45s

#### 3.1 视频叙事结构（测评5段式）
1. 实测钩子【0-5s】：桌面全景，多款产品平铺对比
2. 产品核验【5-12s】：配料表怼拍、参数特写
3. 质地口感实测【12-28s】：勺子舀取、拉丝特写
4. 横向对比总结【28-38s】：多品同框对比
5. 理性收尾【38-45s】：温和参考式种草

#### 3.2 镜头范式
- 光线：室内日常自然光，无滤镜无美化
- 机位：固定平视桌面机位、产品特写、配料表怼拍
- 运镜：固定镜头为主，无复杂运镜

#### 3.3 表达范式
1. 全程真人快速口播，语言口语化
2. 字幕只高亮核心参数，不堆砌文字
3. 杜绝绝对化用语、夸大话术
4. 保持有优有劣的客观口吻

### 4. 产品植入规则
1. 产品出镜控制在2-3次
2. 以多品横向测评对比形式植入
3. 卖点只讲配料、糖度、蛋白、口感
4. 无强引导下单话术
5. 结尾温和参考式种草

### 5. 选择理由
1. 视频结构极度标准化，适配测评模板
2. 和豆豆子风格完全差异化
3. 博主风格本身克制真实，天然规避违规词


## A2 AI 工作流与 Skill 设计

### 1. 工作流设计 (5步 Agent 流程)

```
Step 1: 需求结构化解析          Step 2: 达人内容风格建模
    │                                  │
    └──────────────┬───────────────────┘
                   ▼
    Step 3: 商单脚本标准化生成
          │                    ← ─ ─ ┐
          ▼                          │
    Step 4: 内容合规风控校验          │ 迭代修复
     ┌───────┴───────┐              │ (最多3次)
     │ PASS          │ FAIL ─────────┘
     ▼               ▼
    Step 5: 脚本归档 (飞书/本地)
```

### 2. 各步骤 Prompt 设计

#### Step 1: Brief 解析 Prompt
```
系统角色：MCN 品牌 Brief 解析助手
任务：从品牌 Brief 文本中提取结构化信息
输出格式：JSON（brand_name, product, flavors, core_selling_points, 
          target_audience, usage_scenarios, constraints）
约束：只返回 JSON，不包含其他内容
```

#### Step 2: 风格分析 Prompt
```
系统角色：小红书博主内容风格分析助手
任务：从博主内容拆解报告中提取结构化风格特征
输出格式：JSON（blogger_name, persona, scenes, expression, 
          camera_style, narrative_structure, product_placement_rules）
```

#### Step 3: 脚本生成 Prompt
```
系统角色：小红书短视频脚本生成助手
约束：
1. 严格遵守参考博主风格（无口播、字幕分段、舒缓BGM）
2. 产品自然植入作为场景化食材
3. 禁止功效承诺用语
4. 字幕短句，2行以内
5. 时长50-70秒
6. 5段式叙事：开场钩子→食材展示→分步制作→成品摆盘→结尾
输出格式：Markdown 表格分镜表
```

#### Step 4: 合规检查 Prompt
```
系统角色：内容合规审查专家
检查维度：
- 功效承诺（含暗示性话术）
- 夸大宣传/绝对化用语
- 医疗暗示/引导购买
- 风格匹配度
- 植入自然度
输出格式：JSON（checks数组 + suggestions数组）
```

### 3. 质检规则

| 检查项 | 方法 | 一票否决？ |
|--------|------|-----------|
| 功效承诺词 | 规则匹配 | 是 |
| 夸大用语 | 规则匹配 | 是 |
| 医疗暗示 | LLM语义 | 是 |
| 风格匹配 | 规则+LLM | 否（建议修改）|
| 植入自然度 | 规则+LLM | 否（建议修改）|
| 时长控制 | 规则解析 | 否（建议修改）|

### 4. 输入输出格式

**输入**:
- 品牌 Brief（自由文本）
- 博主内容拆解（自由文本，来自 A3 部分）
- 场景主题（字符串，如"夏日早餐"）

**输出**:
- 结构化 Brief（JSON）
- 风格配置（JSON）
- 完整短视频脚本（Markdown，含分镜表）
- 合规检查报告（JSON + 自然语言）
- 飞书文档 URL / 本地 Markdown 文件
